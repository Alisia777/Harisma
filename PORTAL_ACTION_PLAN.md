# Пошаговый план доработки портала Алтея

## 1. Что уже закрыто в v4

- вынесен отдельный экран **Контроль задач**;
- добавлен визуальный **свод задач** и **чек-лист контроля**;
- упрощён реестр SKU, чтобы он читался как рабочий список;
- карточка SKU стала рабочим центром: задачи, апдейты, owner, причины фокуса;
- добавлены авто-сигналы по марже, остаткам, возвратам, owner и внешнему трафику.

## 2. Что делать следующим приоритетом

### Шаг 1. Сделать единый боевой задачник

Что нужно:
- вынести задачи из локального браузера в общую базу;
- сохранить поля:
  - SKU
  - площадка
  - тип задачи
  - owner
  - срок
  - статус
  - следующее действие
  - история апдейтов

Рекомендуемый стек:
- **Supabase** для таблиц `tasks`, `task_updates`, `sku_comments`, `profiles`;
- авторизация через email magic link или Google.

### Шаг 2. Ввести роли

Минимум 4 роли:
- **viewer** — только просмотр;
- **owner** — может обновлять свои задачи;
- **lead** — может назначать owner и менять приоритеты;
- **admin** — управляет настройками и порогами.

### Шаг 3. Вынести пороги из кода

Сейчас логика "в работу" и авто-сигналы сидит в коде. Лучше вынести в отдельный JSON / таблицу:
- порог выполнения плана;
- порог отрицательной маржи;
- порог low stock;
- порог возвратов;
- правило фокусных SKU без внешнего трафика.

### Шаг 4. Автоматизировать загрузку рабочих файлов

Текущая схема:
- Excel / CSV → `build_data.py` → JSON → сайт.

Следующий уровень:
- рабочие файлы в Google Drive / OneDrive / SharePoint;
- nightly job или кнопка "Обновить данные";
- авто-пересборка JSON в GitHub Actions или backend-функции.

### Шаг 5. Доделать контроль новинок

Нужен отдельный gate-board:
- идея;
- аналитика;
- PMR;
- дизайн;
- тест;
- поставка;
- карточка;
- контент;
- внешний трафик;
- first review.

### Шаг 6. Привязать weekly / monthly / PMR к задачам

Логика должна стать сквозной:
- **PMR** → статусы SKU / новинки / вывод;
- **Monthly Marketing Review** → фокусные SKU / трафик / контент;
- **Weekly** → список задач на неделю с owner и сроком.
