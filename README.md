  - широкий слой артикулов в риске по оборачиваемости;

- правки owner:
  - Кирилл → Олеся,
  - добавлен Артем в список owner,
  - `retinol_boost_krem_dlya_vek_30ml` закреплён за Олесей;
- **фиксы читаемости** для select / option в задачах.

## Что остаётся из v6

- командный режим с Supabase-ready слоем;
- статус синхронизации в topbar;
- загрузка / отправка командных данных;
- задачи, комментарии, решения, owner assignment;
- дашборд, документы, репрайсер, реестр SKU, новинки, executive view.

## Новые файлы в v7

- `imperial.css` — визуальный слой и фиксы select / option
- `app-imperial.js` — оверлей с owner-миграцией, графиками и логистикой
- `data/platform_trends.json` — дневной pulse по площадкам
- `data/logistics.json` — кластера, склады, backlog, риск-слой
- `scripts/build_imperial_layer.py` — пересборка imperial-слоя из рабочих файлов

## Как обновлять Imperial-слой

```bash
python scripts/build_imperial_layer.py --repo-dir .
```

Скрипт:

- патчит owner-слой;
- пересобирает `platform_trends.json`;
- пересобирает `logistics.json`;
- обновляет `skus.json` и `seed_comments.json`.

## Как включить командную синхронизацию

1. Создай проект в Supabase.
2. Выполни `backend/supabase_schema.sql`.
3. Заполни `config.js`.
4. Поставь `teamMode: 'supabase'`.
5. Если уже есть локальные задачи — нажми **«Отправить в командную базу»**.

## Ограничения текущей версии

- график маржи по дням — **операционная оценка**, а не закрытая финансовая отчетность;
- live-подключение Google Sheets / OneDrive пока не встроено;
- права и роли пока не разграничены на уровне UI;
- часть логистики WB пока идёт по складовому слою, без кластера локальности как у Ozon.

## Следующий слой после v7

1. live-подключение рабочих файлов;
2. отдельный логистический экран с фильтрами по owner / кластеру / SKU;
3. дневная финансовая маржа из первички по площадкам;
4. роли по рабочей почте;
5. уведомления и история изменений.
