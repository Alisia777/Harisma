# Altea Portal · GitHub Pages MVP v4

Обновлённый портал Алтея под развёртывание на GitHub Pages.

## Что добавлено в v4

- отдельный экран **Контроль задач**;
- визуальный **чек-лист контроля**;
- kanban-свод по статусам задач;
- упрощённый реестр SKU: статус → owner → внешний трафик → следующее действие → дедлайн;
- авто-сигналы по SKU:
  - план < 80% + отрицательная маржа;
  - низкий остаток;
  - высокий возврат;
  - нет owner;
  - фокусный SKU без внешнего трафика;
- карточка SKU с задачами, апдейтами и ручным созданием задач;
- белые бейджи артикулов сохранены.

## Что уже есть

- обзор бренда Алтея;
- реестр SKU только по Алтея с явным фильтром WB / Ozon;
- карточка SKU с план/факт, отрицательной маржой, остатками, возвратами и owner;
- иконки внешнего трафика КЗ / VK;
- календарь новинок;
- экран рабочего ритма weekly / monthly / PMR;
- локальные комментарии и задачи по артикулу;
- owner по артикулам из рабочих реестров.

## Что важно понимать сразу

Эта версия рассчитана на **быстрый старт на GitHub Pages**.

GitHub Pages хорошо подходит для:

- статического фронта;
- публикации дашбордов и внутренних порталов без сервера;
- быстрого прототипа до покупки домена.

Но GitHub Pages сам по себе **не даёт общего хранилища комментариев**. Поэтому в этой версии комментарии и задачи сохраняются **локально в браузере**. Это удобно для demo / MVP / пилота, но для общей работы команды дальше нужен backend:

- Supabase
- Firebase
- Google Sheets + Apps Script
- Airtable + automation

## Структура проекта

- `index.html` — портал
- `styles.css` — стили
- `app.js` — логика UI
- `data/` — готовые JSON для фронта
- `assets/` — иконка
- `scripts/build_data.py` — пересборка JSON из новых выгрузок
- `PORTAL_ACTION_PLAN.md` — пошаговый план, что доделывать дальше
- `SITE_AUDIT.md` — аудит работоспособности портала

## Как выложить на GitHub Pages

### Вариант без build-пайплайна

1. Создай новый репозиторий на GitHub.
2. Залей содержимое этой папки в корень репозитория.
3. Открой repo → **Settings** → **Pages**.
4. В блоке **Build and deployment** выбери:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main`
   - **Folder**: `/(root)`
5. Нажми **Save**.
6. Через короткое время GitHub выдаст ссылку вида:
   - `https://username.github.io/repo-name/`

## Как потом прикрутить свой домен

1. Открой repo → **Settings** → **Pages**.
2. В поле **Custom domain** укажи домен.
3. У регистратора домена добавь DNS-записи под GitHub Pages.
4. После обновления DNS нажми **Check again**.
5. После выпуска сертификата включи **Enforce HTTPS**.

## Как обновлять данные

Сейчас в `data/` уже лежит готовый срез только по бренду Алтея, уже с owner, фильтрами WB/Ozon и отметками внешнего трафика.

Чтобы обновлять портал новыми выгрузками:

1. Положи новые выгрузки локально в отдельную папку.
2. Запусти:

```bash
python scripts/build_data.py --source-dir /путь/к/выгрузкам --output-dir ./data --brand-filter Алтея
```

3. Закоммить обновлённые JSON в репозиторий.

## Текущие ограничения версии MVP

- комментарии и задачи не общие между сотрудниками;
- нет авторизации;
- нет ролей и прав доступа;
- данные публикуются как статические JSON;
- автообновления из кабинетов маркетплейсов пока нет.

## Следующий логичный шаг

1. Поднять shared backend для комментариев и задач.
2. Добавить авторизацию сотрудников.
3. Сделать авто-пересборку JSON по расписанию.
4. После этого — подключить домен, backend и общий контур командной работы.
