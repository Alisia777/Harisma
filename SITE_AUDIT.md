## Что уже работает как MVP

### 1. Обзор бренда
- KPI по Алтея;
- блок `Что требует внимания сегодня`;
- быстрые кнопки перехода в контроль.

### 2. Контроль задач
- kanban по статусам;
- чек-лист контроля;
- фильтры по owner / типу / площадке / горизонту / источнику;
- авто-сигналы и ручные задачи;
- визуальная проверка просрочки и критичности.

### 3. Реестр SKU
- только Алтея;
- WB / Ozon фильтр;
- owner;
- внешний трафик;
- рабочий статус;
- следующее действие и срок.

### 4. Карточка SKU
- план / факт;
- маржа WB / Ozon;
- остатки и возвраты;
- задачи по SKU;
- ручное создание задач;
- комментарии / апдейты.

### 5. Новинки и ритм работы
- pipeline новинок;
- экран с weekly / monthly / PMR.

## Ограничения текущей версии

### Главный стоп-фактор для командной работы
Комментарий и задача пока живут **локально в браузере**. Значит:
- у одного пользователя апдейт виден;
- у другого — нет;
- это ещё не общий боевой портал.

### Ограничение по данным
Сайт работает на заранее собранных JSON, а не на live-подключении к рабочим файлам.

## Итоговая оценка

### Как управленческий MVP
**Хорошо**.
Портал уже можно использовать как:
- дашборд владельца / бренд-лида;
- рабочий реестр SKU;
- визуальный контроль week-to-week задач.

### Как командный портал
**Пока частично**.
Для полноценной ежедневной командной работы не хватает:
- общей базы задач;
- авторизации;
- истории изменений;
- общей синхронизации комментариев.

## Следующий реальный шаг

1. Поднять backend для задач и комментариев.
2. Подключить источник рабочих файлов.
3. Сделать автообновление JSON.
4. После этого уже добивать домен и полный боевой режим.
