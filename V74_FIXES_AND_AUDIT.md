# V7.4 fixes and audit

## Что было сломано

1. **Синк падал с `platformBadge is not defined`**.
   Причина: в слое `app-v71.js` использовалась функция `platformBadge`, которой не было в runtime.
   Из-за этого `rerenderCurrentView()` ломался в момент рендера задач, а при pull/push создавалось ощущение, что не работает Supabase.

2. **Логистика была неудобной для разбора**.
   Был master-detail сценарий `точка → список SKU → правая панель`, но не было широкого табличного вида как на рабочем разборе.

3. **В логистике показывались только risk rows**.
   Если по выбранному кластеру не было SKU в зоне риска, экран выглядел пустым, хотя в точке товары были.

4. **Лидеры на дашборде открывались только по кнопке-артикулу**.
   Из-за этого ряд воспринимался как некликабельный.

## Что исправлено

### Sync / tasks
- добавлен совместимый helper `platformBadge(platform)` через `taskPlatformBadge`;
- задача больше не ломает общий рендер;
- pull/push синка перестают падать на этой ошибке.

### Dashboard usability
- строки лидеров на дашборде сделаны кликабельными целиком;
- при клике по строке открывается карточка SKU.

### Logistics 2.0
- логистический экран переделан в **широкую таблицу**;
- добавлены переключатели **WB / Ozon** и цели **7 / 14 / 28 дней**;
- добавлен режим **«Все кластера / все склады»**;
- добавлен поиск по артикулу / названию / owner / точке;
- снизу оставлен детальный блок по выбранному SKU;
- root cause остаётся виден и в строке, и в детальном блоке.

### Data layer
- `logistics.json` теперь содержит `allRows`, не только `riskRows`;
- слой собран из реальных источников:
  - Ozon stocks report,
  - Ozon orders,
  - WB supplier goods,
  - central warehouse.

## Что ещё стоит проверить после выката
1. Создать задачу в браузере A.
2. Нажать `Синхронизировать изменения`.
3. В браузере B нажать `Обновить командные данные`.
4. Проверить, что задача приехала.

Если после этого синк всё ещё не едет, тогда уже смотреть именно сетевой ответ Supabase / RLS, а не фронтендовый рендер.
