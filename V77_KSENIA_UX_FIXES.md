# v7.7 — UX-фикс блока «Продукт / Ксения»

Что изменено:

1. Убраны две большие формы сверху.
   - Товар теперь редактируется прямо в строке списка.
   - Новинка теперь создаётся и редактируется прямо в карточках списка.

2. Исправлена читаемость.
   - Белые карточки и инпуты переведены в тёмный контрастный режим.
   - Селекты, textarea и строки редактирования читаются на тёмном фоне.

3. Добавлены сворачиваемые блоки.
   - Как работать в блоке Ксении
   - Реестр товаров
   - Карточки новинок
   - Задачник Ксении
   - Тетрадь действий

4. Добавлены пояснения по назначению блоков.
   - зачем нужен блок
   - что в нём хранится
   - как им пользоваться
   - где создавать новый товар / новинку

5. Починена логика действий в реестре.
   - «Править в списке» переводит строку в режим inline-редактирования
   - «Сохранить изменения» реально сохраняет товар
   - «Удалить» спрашивает подтверждение перед удалением

6. Починена логика действий по новинкам.
   - новая новинка добавляется кнопкой в заголовке блока
   - карточка новинки редактируется прямо в сетке карточек
   - удаление новинки идёт с подтверждением

7. Сохранён общий sync-контур через Supabase.
   - товары / новинки / тетрадь действий продолжают использовать тот же data layer
   - схема v7.6 остаётся валидной, отдельная новая SQL migration не нужна
