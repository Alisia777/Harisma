# V7.8 fix

Что исправлено:
- добавлена отсутствующая функция `v76ManagerSummary`, из-за которой падал блок Ксении;
- `rerenderCurrentView()` теперь рендерит каждый экран изолированно и не валит весь портал одной ошибкой;
- статус sync теперь показывает, если база online, но у UI есть ошибка;
- блок `Продукт / Ксения` и `Ритм работы` не должны оставаться пустыми из-за одной фронтовой ошибки.

Почему это происходило:
- при первом ререндере портал падал на вкладке Ксении из-за `v76ManagerSummary is not defined`;
- после этого `renderMeetings()` уже не вызывался, поэтому `Ритм работы` тоже был пустой;
- зелёный статус сверху показывал только состояние командной базы, а не факт успешной отрисовки экрана.
