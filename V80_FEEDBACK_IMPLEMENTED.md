# Altea Portal v8.0 — feedback implementation

Что включено:

1. Блок «Продукт / Ксения» больше не обнуляется, если в Supabase пустые таблицы товаров/новинок, а тетрадь действий непустая.
2. Для товаров и новинок добавлены продуктовые поля:
   - месяц запуска план / факт
   - МРЦ
   - максимальная скидка
   - план выручки + период
   - план, шт.
   - инвестиции (разработка / первая партия / прочее)
   - break-even (шт. / мес.)
   - ROI
   - риск
   - ссылки на ТЗ / рецептуру / дизайн / договор
3. Реестр товаров Ксении редактируется прямо в таблице.
4. Карточки новинок поддерживают запуск launch checklist по шаблону `Копия Чек-лист запуска новинок.xlsx`.
5. В карточке SKU добавлена кнопка «Скопировать SKU».
6. В блоке «Руководителю» summary-плитки стали раскрывать детальный drill-down по задачам / SKU.
7. Загрузка портала не блокируется ожиданием sync — сначала рендер локально, потом подключение к базе в фоне.

Важно:
- для синка новых полей товаров/новинок в Supabase нужно один раз прогнать `backend/supabase_schema_v80_product_payload.sql`.
- без миграции портал остаётся рабочим, но новые поля хранятся локально.
