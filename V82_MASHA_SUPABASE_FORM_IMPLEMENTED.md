# V8.2 · Машина форма статусов перенесена в Supabase

Что сделано:
- вместо file-source добавлен полноценный Supabase-backed контур `portal_masha_status_updates`;
- форма статусов WB встроена в портал в двух местах:
  - внутри карточки SKU;
  - в блоке «Задачи» как отдельный рабочий стол РОПа;
- текущий статус по WB показывается в реестре SKU;
- в карточке SKU видны:
  - текущий статус,
  - риск,
  - следующий шаг,
  - срок,
  - история обновлений;
- запись статуса сохраняется локально и, если Supabase доступен, сразу синхронизируется в базу;
- `pull/push` кнопки портала теперь тянут и выгружают WB-статусы тоже;
- file-source `data/masha_status_updates.json` оставлен как seed/fallback, но не как основной рабочий источник;
- action-item'ы из формы по-прежнему не дублируют весь board: задача создаётся только из текущих actionable статусов.

Что нужно для боевого режима:
1. прогнать `backend/supabase_schema_v82_masha_status_form.sql`;
2. выкатить портал;
3. проверить сценарии:
   - новая запись WB-статуса из карточки SKU;
   - новая запись WB-статуса из блока «Задачи»;
   - редактирование существующей записи;
   - удаление не-seed записи;
   - pull/push статусов между двумя браузерами.

Что пока оставлено за рамками:
- bulk import из xlsx в браузере;
- отдельная RLS-модель по ролям РОП/руководитель;
- мультиплощадочная форма для OZ / ЯМ — структура готова, UI пока WB-first.
